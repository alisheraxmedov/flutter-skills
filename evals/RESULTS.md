# Eval results — what the skills measurably change

Generated by `evals/run.sh` (baseline vs with-skill, LLM-judged). Raw per-run
output lands in `evals/results/` (gitignored, regenerated each run). Reproduce
with the commands below.

## Headline (measured)

On the 7-case high-signal suite, the skills lift a **cost-efficient model
(Haiku 4.5)** from **38% → 64%** (baseline → with-skill) — a **+26 pp** uplift,
**mean of 2 runs**. That is where the value shows up in these evals.

On a **frontier model (Opus 4.8)** the with-skill score is ~100%, but the
*baseline is already near-ceiling* on single-file snippets — so the **measurable
delta there is ≈ 0**. The skills' frontier value is cross-file architecture and
consistency on large projects, which this single-file snippet suite does **not**
measure (see Notes). We do not claim a frontier-model uplift from these evals.

| Model | Baseline | With-skill | Measured delta |
|---|:--:|:--:|:--:|
| Haiku 4.5 (cost-efficient) | 38% | **64%** | **+26 pp** (mean of 2 runs) |
| Opus 4.8 (frontier) | ~100% | ~100% | ~0 (already near-ceiling here) |
| Sonnet 4.6 | not yet measured | not yet measured | — |

Numbers are noisy at this sample size — treat them as directional, not precise.

## Where the skills add the most value

- **Cost-efficient models (Haiku / Sonnet).** This is where the measured uplift
  lives — the skill pulls a cheaper, faster model up toward senior-grade Flutter
  output. If you run a cheaper model to save cost, that is where they pay off.
- **Large, real, multi-file projects.** The architecture rules, file-placement,
  orchestration and output contract are *designed* to compound across a codebase.
  This is the skills' intended core value — but it is **not measured here**;
  single-file snippet evals under-represent it. Treat it as design intent, not a
  benchmarked claim, until we add multi-file evals.
- **Release / native / security tasks.** The most consistent per-task wins (e.g.
  release-safe `dio` logging, secure token storage + Dart-side TLS pinning).

## Method

7 high-signal cases (`evals/evals-hard.json`): the Riverpod rebuild bug plus the
release/native/security tasks (persistence package choice, flavors, FCM background
handler, token storage, release-safe logging). Each case is generated twice —
clean **baseline** and **with-skill** (loaded via `--plugin-dir`) — then scored by
a neutral grader against the case's objective `expected_behavior` /
`anti_behaviors`. A case passes only when every expected item holds and no
anti-behaviour appears. Empty generations and unparseable judge replies are
reported as **ERR and excluded** — never scored 0% or 100%.

```bash
# Cost-efficient model (the measured uplift):
MODEL=claude-haiku-4-5-20251001 EVALS_FILE=evals/evals-hard.json ./evals/run.sh

# Your default (frontier) model:
EVALS_FILE=evals/evals-hard.json ./evals/run.sh
```

## Per-run detail (cost-efficient model, Haiku 4.5)

| run | baseline | with-skill | uplift | cases helped |
|---|:--:|:--:|:--:|:--:|
| run 1 | 23% | 57% | +34 pp | 3/7 |
| run 2 | 52% | 71% | +19 pp | 2/7 |
| **mean** | **38%** | **64%** | **+26 pp** | — |

The largest single-task swing was on **networking** (the baseline leaked auth
headers via an unguarded `LogInterceptor` in release; the skill guards it with
`kDebugMode`). Per-task numbers on a single run are noisy — that swing should not
be read as a stable per-task figure.

## Notes for the reader (read these before quoting a number)

- **n = 2 runs.** Generation and the LLM judge are both non-deterministic;
  baseline alone moved 23% → 52% between runs. The **mean uplift on the
  cost-efficient model** is the figure to trust; single-run and single-task
  numbers are noise. **Average 3–5 runs** before quoting anything.
- These are **objective-assertion** evals on **single-file snippets** — they
  capture per-task correctness, not the large-project architecture/consistency
  value that is the skills' main intended strength. That value is **unmeasured
  here**, not proven.
- **Opus delta ≈ 0 is expected and stated on purpose** — the frontier baseline is
  already near-ceiling on these snippets. We don't bury it.
